III. Descripción del examen:
El examen consiste en Presentar un producto digital en un portafolio personal utilizando las
herramientas tecnológicas y buenas prácticas disciplinares para resolver una problemática y
potenciar el perfil profesional.
El producto digital principal será la creación de un blog de noticias. Dicho Blog debe contemplar los
requerimientos entregados durante las clases. Dichos requerimientos estarán alineados con las
siguientes competencias del plan formativo del curso:
 Implementar páginas web básicas responsivas utilizando HTML, CSS y JavaScript de acuerdo
con los requerimientos y acorde a las buenas prácticas de la industria.
 Codificar piezas de software de baja/mediana complejidad utilizando lenguaje Javascript para
resolver problemáticas comunes de acuerdo con las necesidades de la industria.
 Codificar piezas de software de baja/mediana complejidad en lenguaje Javascript utilizando
paradigmas de orientación a objetos, orientación a eventos y programación asíncrona para
resolver problemáticas de acuerdo con las necesidades de la organización.
 Operar una base de datos relacional utilizando el lenguaje SQL para la obtención,
manipulación y definición de datos dando solución a un problema de almacenamiento de
información.
 Desarrollar aplicaciones web utilizando el entorno multiplataforma Node Express para dar
solución a los requerimientos de la organización
 Desarrollar una aplicación que ejecute operaciones de consulta y manipulación de datos
utilizando el entorno Node.js de acuerdo con las buenas prácticas de la industria.
 Implementar aplicaciones empresariales que disponibilizan servicios Rest utilizando el
framework Express y el entorno Node.js para dar solución a los requerimientos de la
organización.

IV. Consideraciones para los participantes:
El examen es de ejecución individual, es decir, debe ser resuelto solamente por usted, no puede
intercambiar información con sus compañeros ni con personas externas. Sin embargo, usted puede

consultar internet libremente si lo requiere.

 Tener PG admin 4 instalado en su equipo

PROYECTO: BLOG DE NOTICIAS
IV. Descripción:
Los alumnos del curso, Desarrollo de aplicaciones Full Stack JavaScript Trainee, deben realizar un
proyecto final de portafolio; para cumplir con esto, los alumnos deberán crear un “BLOG DE
NOTICIAS”
El blog debe tener a lo menos lo siguientes requerimientos:
 Registro/Login de Usuarios.
 Postear Noticias.
 Escribir comentarios.
 Sistema de likes
 El usuario puede ver el número de likes de una noticia y quienes dieron like
Los post de noticias deben cumplir a lo menos con los siguientes requerimientos:
 Título de la Noticia. Máximo 60 caracteres
 Imagen de la Noticia. Tamaño Máximo 2mb..
 Texto de la noticia. Máximo 4000 caracteres.
 Número de Likes y Dislikes.
 Listado de usuarios que dieron like y número de usuarios que dieron dislike. Esto puede
manejarse de forma distinta según el módelo de datos a utilizar.
 Fecha de la Noticia. En formato Date.
 Categoría de la Noticia. Se sugiere trabajarlo en una tabla independiente y Relacionada.
 Autor de la Noticia. Máximo 40 caracteres.

a) Las noticias y los datos de usuarios deben estar en una base de datos y se debe acceder a estos
usando un backend creado en node/express.
b) La página debe estar alojada en un hosting estático y el backend en un servicio cloud. El frontend
del sitio de noticias debe maquetarse usando Bootstrap.
c) La página de inicio debe tener una barra de navegación por categorías, un footer, un hero y las
noticias deben estar ordenadas desde la última noticia hacia atrás en el tiempo. Este orden,
filtrado y selección de noticias debe hacerse en el frontend/backend y en la base de datos. Se

puede usar de forma opcional un ORM para trabajar con los datos. Las categorías con noticias
deben permitir ordenar las noticias por antigüedad o invertir ese orden.

d) Al clickearse el encabezado de una noticia en el frontend, el usuario debe poder navegar al
detalle de la noticia en otra página.
e) Cada elemento de la programación del blog, en base de datos, backend y frontend, debe seguir
lo siguiente:
 Utilización general del lenguaje, sintaxis, selección de tipos de datos, sentencias lógicas,
expresiones, operaciones, comparaciones
 Utilización de sentencias repetitivas
 Convenciones y estilos de programación

V. Consideraciones:
 El portafolio se va a alojar en github.
 Se va a desplegar una web utilizando github.
 Se realizará un despliegue de backend en un servicio cloud. Para ese despliegue hay un plazo
de 24 horas, pero el código fuente a desplegar debe entregarse al terminar la prueba en 6
horas o menos.
 Se explicará el proceso por el alumno a través de un video en YouTube.
 El producto digital final debe ser visto en la web. Se puede alojar el sitio en github pages y el
backend en un sistema cloud gratuito o alojar todo en un mismo servidor cloud gratis.
 Se debe realizar un video en youtube explicando el funcionamiento y la forma en que se creó
el producto digital final.
 Aunque se sugiere que se creen backends independientes para login/registro de usuarios y
compra de productos, para mostrar la idea de microservicios, se puede realizar también el
proyecto utilizando solo 1 aplicación de backend.
